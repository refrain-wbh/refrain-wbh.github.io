The Verification Horizon: No Silver Bullet for Coding Agent Rewards

Binghai Wang et al. · Qwen Team Technical Report · arXiv:2606.26300

"There is no silver bullet."
— Frederick P. Brooks, Jr., No Silver Bullet: Essence and Accident in Software Engineering (1986)

Abstract. A classical intuition holds that verifying a solution is easier than producing one. For today's coding agents, this intuition is being inverted: as foundation models develop stronger reasoning and engineering harnesses grow more sophisticated, generating complex candidate solutions is no longer difficult—reliably verifying them has become the harder problem. Every verifier we can build is only a proxy for human intent, never the intent itself. This creates a twofold difficulty: intent is inherently under-specified, so faithfully checking whether it is satisfied is extremely hard; and under optimization, the gap between proxy and intent widens, manifesting as reward hacking or signal saturation. We characterize the quality of verification signals along three dimensions—scalability, faithfulness, and robustness—and argue that achieving all three at once is the core challenge. Building on Qwen foundation models, we study four reward constructions: a test verifier for SWE-style tasks, a rubric verifier for frontend tasks, the user as verifier for real-world agent tasks, and an automated agent verifier for long-horizon tasks. Across task types and policy capability levels, targeted verification design suppresses reward hacking and improves task quality, yielding significant gains on internal and public benchmarks. No fixed reward function can remain effective as policy capability keeps growing; verification must co-evolve with the generator—a continually rebuilt reward system is the core infrastructure that turns raw capability growth into trustworthy capability growth.
